# **Каким образом в реляционных СУБД можно задать правила, которым должны соответствовать данные в таблице? Назовите известные вам типы правил, приведите примеры использования.**

### В реляционных БД правила задаются ограничениями(**constraints**), работает как фильтр.

#### Примеры типов правил:

* NOT NULL (запрещает оставлять поле пустым)
* UNIQUE (уникальность. гаранитрует что в столбце будут неповторяющиеся значения)
* CHECK (проверка условий. проверяет вводимые данные по заданому заранее логическому условию (чето типо if else))
* DEFAULT (по умолчанию. подставляет заранее заданное значение если оно не было введено)
* PRIMARY KEY(первичный ключ. уникальный индификатор который обьеденяет в себе )
* FORIGN KEY(внешний или вторичный ключ. связывает между собой колонки по первичному ключу)
