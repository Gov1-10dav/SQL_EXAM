### 12. Операции над множествами в SQL.
SQL поддерживает три основные операции над множествами (результатами запросов), которые работают с **совместимыми по структуре** таблицами (одинаковое число столбцов и совместимые типы):

#### 1. `UNION` — объединение

- Объединяет результаты двух запросов **без дубликатов**.
- Аналог математического объединения множеств.
- Пример:
    sql
    `SELECT name FROM Students`
	`UNION`
	`SELECT name FROM Teachers;`

#### 2. `UNION ALL` — объединение с сохранением дубликатов

- Объединяет результаты, **сохраняя все строки**, включая полные дубликаты.
- **Применение:** Используется гораздо чаще, чем простой `UNION`, так как работает значительно быстрее (СУБД не тратит ресурсы на проверку и удаление повторов).

#### 3. `INTERSECT` — пересечение

- Возвращает только те строки, которые **присутствуют в обоих запросах**.
- Пример:
    sql
    `SELECT product_id FROM Orders_2024`
	`INTERSECT`
	`SELECT product_id FROM Orders_2025;`
    

#### 4. `EXCEPT` (в некоторых СУБД — `MINUS`) — разность

- Возвращает строки из первого запроса, **которых нет во втором**.
- Пример:
    sql
    `SELECT customer_id FROM All_Customers`
	`EXCEPT`
	`SELECT customer_id FROM VIP_Customers;`
    

> Все три операции **автоматически удаляют дубликаты**. Чтобы сохранить их, в некоторых СУБД можно использовать `UNION ALL`, но `INTERSECT ALL` и `EXCEPT ALL` поддерживаются не везде (например, в PostgreSQL — да, в MySQL — нет).
