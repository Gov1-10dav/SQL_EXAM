### 11. Вложенные запросы. Классификация.
---
**Вложенный (подчинённый) запрос** — это `SELECT`, размещённый внутри другого SQL-запроса (в `SELECT`, `FROM`, `WHERE` или `HAVING`).
#### Классификация вложенных запросов:

#### 1. По количеству возвращаемых значений

- **Скалярный подзапрос** — возвращает **одно значение** (одна строка, один столбец).  
    Пример:
    sql
    `SELECT name FROM Students` 
	`WHERE age = (SELECT MAX(age) FROM Students);`
    
- **Подзапрос с множеством значений** — возвращает **один столбец, несколько строк**. Используется с операторами `IN`, `ANY`, `ALL`.  
    Пример:
    sql
    `SELECT name FROM Students` 
	`WHERE group_id IN (SELECT id FROM Groups WHERE course = 'Math');`
---
#### 2. По зависимости от внешнего запроса

- **Независимый (некоррелированный)** — может выполняться отдельно, не использует данные из внешнего запроса.
- **Коррелированный** — зависит от значений из внешнего запроса и выполняется **для каждой строки** внешнего запроса.  
    Пример:
    sql
    `SELECT name FROM Students s` 
	`WHERE EXISTS (SELECT 1 FROM Grades g WHERE g.student_id = s.id AND g.score < 3);`
---
#### 3. По месту использования

- В `WHERE` / `HAVING` — для фильтрации строк.
- В `FROM` — как **производная таблица** (требует псевдонима).
- В `SELECT` — обычно скалярный подзапрос для добавления вычисляемого столбца.
