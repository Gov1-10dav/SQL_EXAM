# **20.Использование команд SQL в пользовательских функциях**

В PostgreSQL пользовательские функции (создаются через `CREATE FUNCTION`) могут содержать SQL-команды внутри тела функции:

- **SQL-функции** (LANGUAGE sql) — состоят из последовательности SQL-выражений (SELECT, INSERT, UPDATE, DELETE), последнее из которых возвращает результат.

```sql
CREATE FUNCTION get_total_orders(user_id int) RETURNS bigint AS $$
    SELECT COUNT(*) FROM orders WHERE customer_id = user_id;
$$ LANGUAGE sql;
```

- **PL/pgSQL функции** (LANGUAGE plpgsql) — позволяют использовать SQL-команды внутри процедурной логики (циклы, условия, переменные). SQL-команды выполняются через прямые вызовы или `EXECUTE` (для динамического SQL).

```sql
CREATE FUNCTION update_status(oid int) RETURNS void AS $$
BEGIN
    UPDATE orders SET status = 'processed' WHERE order_id = oid;
END;
$$ LANGUAGE plpgsql;
```

**Ограничения и особенности:**

- В функциях, отмеченных как `IMMUTABLE`/`STABLE`, нельзя выполнять команды, изменяющие данные (INSERT/UPDATE/DELETE) — допускаются только в `VOLATILE`.
- Динамический SQL (когда текст запроса формируется как строка) выполняется через `EXECUTE` — нужен для случаев, когда имена таблиц/столбцов известны только во время выполнения. Важно избегать SQL-инъекций, используя `format()` с `%I`/`%L`.
- Функции могут возвращать как скалярные значения, так и наборы строк (`RETURNS SETOF` или `RETURNS TABLE`), что позволяет использовать их в FROM-выражении как таблицу.
- Транзакционность: до PostgreSQL 11 функции выполнялись в рамках одной транзакции без COMMIT внутри; с появлением **процедур** (`CREATE PROCEDURE`) и `CALL` появилась возможность управлять транзакциями (COMMIT/ROLLBACK) внутри тела.
